How To

You asked. We answered. If your question isn't already answered. Reach out on Discord

Command Line

Installation

install lemon globally with npm

> npm install -g lemonjs-cli

Development

Run a local dev server with live-reloads on your src and data

> mkdir your-project
> cd your-project
> lemon new
> lemon dev

Build

This will build your site into the /build directory

> cd your-project
> lemon build

Run

This will host your site as a static site from the build directory

> cd your-project
> lemon run

Deploy to Google Cloud Storage

This will deploy your site to a google cloud storage bucket.

  1. Purchase your domain name (you can use http://domains.google)
  2. Set a CNAME on www.yourdomain.com to c.storage.googleapis.com.
  3. Create a bucket with your domain name (www.yourdomain.com)
  4. Create a service account
    • Service Account Name: deploy
    • Role: Project > Owner
    • Key Type: JSON
  5. Update your lemon.cson
{
  deploy: {
    service: "gcs"
    project: "your-project-id"
    bucket: "www.yourdomain.com"
    credentials: "/absolute/path/to/creds"
  }
}
> cd your-project
> lemon build
> lemon deploy

Deploy to Github Pages

  1. go to your settings page (ex: https://github.com/lemon/lemonjs-cli/settings)
  2. change "Source" to "master branch /docs folder". click "Save"
  3. enter a custom domain. click "Save"
  4. CNAME your domain to YOUR-GITHUB-USERNAME.github.io (more info here]
  5. Update your lemon.cson
{
  deploy: {
    service: "github"
  }
}
> cd your-project
> lemon build
> lemon deploy

Components

Component Namespacing

Every component requires a "package" and a "name". "package" is a namespace for your project. For a website, convention is to use "site" as your package name. For 3rd party libraries, try to keep them short, but easy to remember. For example, lemon ui library uses "lui". The lemon material ui library uses "mui". 

module.exports = lemon.Component {
  package: "site"
  name: "MyComponent"
}

Component Classes

Sometimes you want a component to always have a specific class name. For example, the lui.Button always adds a .lui-button class name to its element for styling. You can do this on your own components with the "class" property.

module.exports = lemon.Component {
  package: "site"
  name: "MyComponent"
  class: 'my-component'
}

Templates

How to set "id" on an element

module.export = lemon.Component {

  template: ->
    div '#my-id-1'
    div id: 'my-id-2'
    div {id: 'my-id-3'}

}

How to set "class" on an element?

module.export = lemon.Component {

  template: ->
    div '.my-class-1'
    div class: 'my-class-2'
    div {class: 'my-class-3'}

}

How to set a conditional "class" on an element?

module.export = lemon.Component {

  template: (data) ->
    div class: {
      'my-class': data.if_should_have_class
    }

}

How to set attributes on an element?

module.export = lemon.Component {

  template: (data) ->
    img src: '', alt: ''
    img {src: '', alt: ''}

    div 'data-title': '', ->
      'content'

    div {'data-title': ''}, ->
      'content'
}

How to write raw html in my template?

By default, all data is escaped so that we can be sure to generate clean, valid html. If you know you have safe html to write, you can use the raw function.

module.export = lemon.Component {

  template: (data) ->
    code ->
      raw data.html_sample
}

How to write content without an element?

Sometimes you want to write text to your dom at the same level of depth as other elements. This is common when you're using spans to do styling on a subtitle or metadata line. The text function can be used to write this text.

module.export = lemon.Component {

  template: (data) ->
    div '.blog-post', ->
      div '.author', ->
        data.author
      div '.metadata', ->
        text "Published on "
        span '.published-on', ->
          data.published_at
        text " in category"
        span '.category', ->
          data.category
}

My template is crashing says xxx tag not defined?

Not 100% of tags are readily available. Sometimes this is for namespace safety, or maybe it's just not a common tag to use so we didn't include it. If you think a tag is missing that should be added, let us know.

Also, sometimes you're using WebComponents you may have non-standard tags that you're using.

Either way, you can use the tag function to use any tag you want.

module.export = lemon.Component {

  template: (data) ->
    tag 'emoji-rain', {active: true}
    tag 'app-drawer'
}

Data

When should I use "static" data?

Most of the time you will use static. In most apps, a majority of the interface is based on items that don't react to events or data changes. All of the page content, the header, the footer, and sidebar list items of this site are static.

For example, once the sidebar on the left is drawn, it doesn't need to be updated again.

When should I use "dynamic" data?

Sometimes you'll have a component with constantly changing data and you'll want dom elements to react in real-time to your data changes.

Here's an example of lemon-ui's Typewriter component. The state of the component changes while it animates, and the UI updates in real-time to react to these changes is state.

Data Binding (Dynamic Data)

Binding Text

If you have a string value property that you want to react to real-time state changes. you can use text-binding.

module.exports = lemon.Component {

  data: {
    property: 'value'
  }

  template: ->

    div bind: {text: 'property'}

    div _text: 'property'

    div _on: 'property', _template: (property) ->
      text property

}

Binding HTML

If you have an html blob that you want to react to real-time state changes. you can use html-binding.

module.exports = lemon.Component {

  data: {
    code_sample: '<div>hello world</div>'
  }

  template: ->

    div bind: {html: 'code_sample'}

    div _html: 'code_sample'

    div _on: 'property', _template: (code_sample) ->
      raw code_sample

}

Binding href attributes

If you have a url property that you want to react to real-time state changes. you can use href-binding.

module.exports = lemon.Component {

  data: {
    my_href: 'http://www.lemonjs.org'
  }

  template: ->
    a bind: {href: 'my_href'}
    a _href: 'my_href'

}

Binding src attributes

If you have an image url property that you want to react to real-time state changes. you can use href-binding.

module.exports = lemon.Component {

  data: {
    my_image: 'http://www.lemonjs.org/img/test.jpg'
  }

  template: ->
    img bind: {src: 'my_image'}
    img _src: 'my_image'

}

Binding templates

Sometimes you want partial updates, but you want to render something more complex than the value of the field. In that case, you can use template-binding.

module.exports = lemon.Component {

  data: {
    active_tab: 'tab1'
    tab_content: {
      tab1: 'hello world'
    }
  }

  template: ->
    div '.tab-wrapper', _on: 'active_tab', _template: (active_tab, data) ->
      div '.tab-header', ->
        active_tab
      div '.tab-content', ->
        data.tab_content[active_tab]

}

Binding Components

Sometimes you want to bind on not just a template, but a full component with its own event handling and data management. In this case, you can use component-binding.

require '../components/CodeBlock'

module.exports = lemon.Component {

  data: {
    code_sample: '<div>hello world</div>'
  }

  template: ->
    div _on: 'code_sample', _component: site.CodeBlock

}

Binding Lists

If you're managing lists, it's not efficient to re-mount the entire list everytime one item is added or removed from the list. Lemon has list-binding to solve this problem. As your list is modified, the dom will only modify the newly added or removed elements.

require '../components/ListItem'

module.exports = lemon.Component {
  package: 'site'
  name: 'List'

  data: {
    items: []
  }

  template: ->
    div _list: 'items', _component: site.ListItem

}

Event Handling

How do I listen for a click event on an element?

module.exports = lemon.Component {
  package: 'site'
  name: 'ClickTestComponent'

  methods: {
    onClick: (e) ->
      alert 'clicked!'
  }

  template: ->
    div $click: 'onClick', ->
      'click me'
}

Sometimes it may be more convenient to put the event handler inline.

module.exports = lemon.Component {
  package: 'site'
  name: 'ClickTestComponent'

  template: ->
    div {$click: -> alert 'clicked!'}, ->
      'click me'
}

How do I listen for events on the entire component?

you can manually add a listener after mount

module.exports = lemon.Component {
  package: 'site'
  name: 'SomeComponent'

  lifecycle: {
    mounted: ->
      @addEventListener @el, 'click', (e) => @onClick e
  }

  methods: {
    onClick: (e) ->
      console.log e
  }

  template: (data) ->

Routing

Direct Match Routing

Use direct match routing to match a pathname exactly.

module.exports = lemon.Component {
  template: ->
    lemon.Router {
      '/en/how-to': ->
    }
}

Named Parameters

Use named paramter routing to pull fields out of your url.

module.exports = lemon.Component {
  template: ->
    lemon.Router {
      '/video/:id': (match) ->
        # load VideoPage component which loads data from your api
        # with match.params.id
    }
}

Expression Parameters

Often when you want some flexibility in a part of your url, but you don't want to let it be anything. In this case you can use expression paramters.

module.exports = lemon.Component {
  template: ->
    lemon.Router {
      '/(en|zh)/how-to': -> # load how to page in given language
      '/video/([0-9]+)': -> # all video ids should be integers
    }
}

Named Expression Parameters

Usually if you have an expression, you'll want to capture its value to use in your component.

module.exports = lemon.Component {
  template: ->
    lemon.Router {
      '/(en|zh):lang/how-to': -> # load how to page in given language
      '/video/([0-9]+):id': -> # all video ids should be integers
    }
}

Splat Parameters

Named parameters can't match across forward slashes, if you want to do that, you can use splat paramters *, which will match everything to the end of the url

module.exports = lemon.Component {
  template: ->
    lemon.Router {
      '/en/*': -> # match all /en... pages
    }
}

Query String

The querystring is also parsed and available in your routes.

module.exports = lemon.Component {
  template: ->
    lemon.Router {
      '/en/how-to': (match) -> match.query
    }
}

Hash

The url hash is also available in your routes.

module.exports = lemon.Component {
  template: ->
    lemon.Router {
      '/en/how-to': (match) -> match.hash
    }
}

Can I Use

Can I use Jquery?

Of course! Lemon has build in methods for finding elements, listening to events, and modifying the dom. So you probably only need jquery if you want to use jquery plugins. Simply add jquery to your head tag.

index.coffee

module.exports = ->
  doctype 5
  head ->
    script src: 'https://cdnjs.cloudflare.com/ajax/libs/jquery/3.3.1/jquery.min.js'
  body ->
    "Your App"

Can I use Jquery FullPage?

Of course! You can use it manually, or use lemon-ui's built-in Jquery FullPage Component.

option 1

module.exports = ->
  doctype 5
  html ->
    head ->
      script src: 'https://cdnjs.cloudflare.com/ajax/libs/jquery/3.3.1/jquery.min.js'
      script src: 'https://cdnjs.cloudflare.com/ajax/libs/fullPage.js/2.9.7/jquery.fullpage.min.js'
      link {
        rel: "stylesheet"
        type: "text/css"
        href: 'https://cdnjs.cloudflare.com/ajax/libs/fullPage.js/2.9.7/jquery.fullpage.min.css'
      }
    body ->
      div id: "fullpage", ->
        div '.section', -> 'section 1'
        div '.section', -> 'section 2'
        div '.section', -> 'section 3'
        div '.section', -> 'section 4'
      script type: 'text/javascript', """
        $(document).ready(function() {
          $('#fullpage').fullpage();
        });
      """

option 2

require 'lemonjs-lui/FullPage'

module.exports = ->
  doctype 5
  html ->
    body ->
      lui.FullPage {
        sections: [
          (-> text 'section 1')
          (-> text 'section 2')
          (-> text 'section 3')
          (-> text 'section 4')
        ]
      }
